# Governance Context

Decision-making, approval processes, change governance.

---

## Authority Levels

**Level 1:** Immutable (needs C-level approval)
- Architecture locks
- Fundamental principles
- Non-negotiable rules

**Level 2:** Architecture (needs architecture council approval)
- Core specifications
- Service design

**Level 3:** Implementation (code review)
- Service features
- API changes

**Level 4:** Design (design review)
- UI changes
- Visual design

**Level 5:** Operations (team consensus)
- Documentation
- workflows

---

## Decision Process

1. Determine authority level
2. Check source-of-truth-policy.md
3. Get required approvals
4. Document in decision-log.md
5. Update affected contexts
6. Announce to team
